# 📷 가족사진관 MVP 요구사항 명세서

## 📌 문서 목적

본 문서는 가족사진관 MVP 개발을 위한 제품 요구사항을 정의한다.  
가족사진관은 가족들이 사진을 카카오톡으로 반복 전송하지 않고, 하나의 Google Drive 기반 사진 공간에 사진을 올리고 함께 감상할 수 있도록 돕는 웹사이트이다.

본 MVP는 복잡한 회원 시스템, 데이터베이스 기반 분류, AI 자동 분류를 제외하고, 다음의 단순한 사용 흐름에 집중한다.

> 6자리 번호 입력 → 가족 사진방 입장 → 최근 사진 감상 → 폴더 생성 → 모바일 사진 업로드 → 감성 갤러리와 지도 기반 추억 보기

---

## 🎯 제품 목표

가족사진관 MVP의 목표는 사진 관리 시스템을 만드는 것이 아니라, 가족이 함께 찍은 사진을 쉽게 모으고 따뜻하게 감상할 수 있는 공유 공간을 만드는 것이다.

사진은 Google Drive에 저장되며, 웹사이트는 Google Drive의 폴더와 사진을 읽어 모바일 친화적인 갤러리 화면으로 보여준다. 사용자는 별도 로그인 없이 6자리 번호를 입력하여 정해진 가족 사진방에 입장한다.

핵심 가치는 다음과 같다.

- 가족별 사진 공유를 단순화한다.
- 사진을 파일 탐색기처럼 보이지 않게 한다.
- 모바일에서 폴더 생성과 사진 업로드가 자연스럽게 이루어지게 한다.
- 최근 사진을 중심으로 가족의 추억을 감성적으로 보여준다.
- 여행이나 외출 사진은 지도와 함께 다시 볼 수 있게 한다.

## 👨‍👩‍👧‍👦 가족 사진방 구조

MVP에서는 사진방을 3개만 운영한다.

### 우리가족 사진방

대상은 나, 배우자, 자녀이다.  
아이 성장 사진, 일상 사진, 가족 여행, 집에서 찍은 사진을 저장한다.

### 나의 부모님 가족 사진방

대상은 나의 부모님, 형제자매, 우리 가족이다.  
부모님과 함께한 식사, 생신, 어버이날, 명절, 가족 모임 사진을 저장한다.

### 처의 부모님 가족 사진방

대상은 처가 부모님, 처가 형제자매, 우리 가족이다.  
처가 가족 식사, 생신, 명절, 가족 여행 사진을 저장한다.

## 🔐 입장 방식

사용자는 사이트 첫 화면에서 6자리 번호를 입력한다.

번호는 총 3개만 사용한다.

```text
123456 → 우리가족 사진방
234567 → 나의 부모님 가족 사진방
345678 → 처의 부모님 가족 사진방
```

번호가 일치하면 해당 사진방으로 이동한다.  
번호가 일치하지 않으면 입장할 수 없다.

로그인은 사용하지 않는다.  
사용자 이름도 입력하지 않는다.  
MVP에서는 번호 자체가 사진방 접근 권한이다.

## 🗂️ Google Drive 폴더 구조

Google Drive에는 최상위 폴더 하나를 만들고, 그 아래에 3개의 사진방 폴더를 둔다.

```text
/family-photo-gallery
  /our-family
  /my-parents-family
  /wife-parents-family
```

각 사진방 안에는 사용자가 직접 폴더를 만든다.

권장 폴더명은 다음 형식을 사용한다.

```text
YYYY-MM-행사명
```

예시는 다음과 같다.

```text
2026-05-어린이날
2026-08-강릉여행
2026-09-추석
2026-12-크리스마스
```

MVP에서는 연도 폴더를 강제하지 않는다.  
사진방 바로 아래에 행사 폴더를 생성하는 단순 구조를 우선한다.

```text
/our-family
  /2026-05-어린이날
  /2026-08-강릉여행
  /2026-12-크리스마스
```

이 방식은 모바일에서 폴더를 만들고 사진을 올리기에 가장 단순하다.

## 🚪 입장 화면 요구사항

입장 화면은 단순하고 조용한 분위기로 구성한다.

### 화면 요소

- 서비스명: 가족사진관
- 짧은 안내 문구
- 6자리 번호 입력 필드
- 입장하기 버튼

### UX 요구사항

- 모바일에서 숫자 키패드가 바로 떠야 한다.
- 입력 필드는 6자리 숫자 입력에 최적화한다.
- 번호 입력이 완료되면 입장 버튼을 누르기 쉽게 표시한다.
- 잘못된 번호를 입력하면 부드럽게 오류 메시지를 보여준다.

### 예시 문구

```text
가족에게 공유받은 6자리 번호를 입력하세요.
우리 가족의 사진방으로 이동합니다.
```

## 📷 사진방 홈 요구사항

사진방 홈은 MVP에서 가장 중요한 화면이다.  
파일 탐색기나 블로그 목록처럼 보이면 안 된다.  
가족 앨범의 첫 장을 여는 느낌이어야 한다.

### 화면 구성 우선순위

- 사진방 이름
- 최근 사진 감성 갤러리
- 사진 올리기 버튼
- 폴더 만들기 버튼
- 디자인된 폴더 목차
- 지도 보기 진입점

### 기본 정렬

사진은 최근 업로드 또는 최근 촬영일 기준으로 보여준다.  
MVP에서는 Google Drive에서 제공 가능한 날짜 정보를 기준으로 최근순 정렬한다.

### 홈 상단 예시

```text
우리가족 사진방
최근에 함께 남긴 사진들
```

또는

```text
나의 부모님 가족 사진방
함께한 날들이 모여 있는 공간입니다.
```

## 🖼️ 최근 사진 갤러리 요구사항

최근 사진 갤러리는 단순 그리드가 아니라 감성적인 배치로 구성한다.

### 지원할 표현 방식

- 큰 대표 사진과 작은 사진 조합
- 폴라로이드 카드 스타일
- 포토북 페이지 스타일
- 벽에 붙인 사진 스타일
- 필름 스트립 스타일
- 잡지형 콜라주 스타일

### 동작 방식

- 최근 사진을 일정 개수 가져온다.
- 사진 개수와 비율에 따라 레이아웃을 선택한다.
- 같은 사진방이라도 화면 진입 시 다양한 배치가 가능하다.
- 사용자는 사진을 눌러 크게 볼 수 있다.

### UX 목표

사용자는 사진 목록을 본다고 느끼기보다, 가족 앨범의 한 장면을 본다고 느껴야 한다.

## 📚 폴더 목차 요구사항

폴더 리스트는 파일 탐색기처럼 보이지 않아야 한다.  
블로그 게시글 목록처럼 보이는 것도 피한다.  
대신 갤러리의 목차, 포토북의 챕터, 전시회의 섹션처럼 보여야 한다.

### 폴더 카드 표현

각 폴더는 하나의 앨범 카드처럼 표현한다.

카드에는 다음 정보를 보여준다.

- 폴더명
- 대표 사진 1장 또는 최근 사진 3장
- 사진 개수
- 최근 업데이트 날짜
- 짧은 보조 문구

### 예시

```text
2026-08-강릉여행
바다 앞에서 남긴 여름의 장면들
사진 42장
```

```text
2026-05-어버이날
함께 식사하고 웃었던 따뜻한 하루
사진 18장
```

### 디자인 방향

- 카드형 앨범 목차
- 큰 여백
- 둥근 모서리
- 부드러운 그림자
- 대표 이미지 중심
- 텍스트는 짧고 따뜻하게

## 📤 모바일 사진 업로드 요구사항

사진 업로드는 모바일에서 가장 많이 사용할 기능이다.  
따라서 PC 파일 업로드 화면처럼 만들면 안 된다.

### 업로드 진입 방식

사진방 홈 하단 또는 우측 하단에 손가락으로 누르기 쉬운 업로드 버튼을 둔다.

권장 형태는 다음과 같다.

- 하단 고정 버튼
- 둥근 플로팅 버튼
- 하단 시트 열기

### 업로드 흐름

```text
사진 올리기 선택
→ 업로드할 폴더 선택
→ 사진 선택
→ 업로드 진행 상태 표시
→ 완료 후 최근 사진 영역 갱신
```

### 모바일 UX 요구사항

- 버튼은 엄지손가락으로 누르기 쉬운 위치에 둔다.
- 업로드 화면은 전체 페이지 이동보다 하단 시트 방식이 적합하다.
- 여러 장의 사진을 한 번에 선택할 수 있어야 한다.
- 업로드 중 진행 상태를 보여준다.
- 완료 후 사용자가 어느 폴더에 올렸는지 알 수 있어야 한다.
- 업로드 실패 시 다시 시도할 수 있어야 한다.

### 업로드 제한

MVP에서는 현재 입장한 사진방 안에만 업로드할 수 있다.  
다른 사진방으로 업로드할 수 없다.

## 📁 모바일 폴더 생성 요구사항

폴더 생성도 모바일에서 자연스러워야 한다.

### 폴더 생성 진입 방식

사진방 홈에서 다음 형태로 제공한다.

- 사진 올리기 버튼 옆에 폴더 만들기 버튼
- 또는 업로드 하단 시트 안에 새 폴더 만들기 제공

### 폴더 생성 흐름

```text
폴더 만들기 선택
→ 폴더명 입력
→ 권장 이름 예시 표시
→ 만들기
→ Google Drive에 폴더 생성
→ 폴더 목차에 반영
```

### 폴더명 안내

사용자가 폴더명을 잘 만들 수 있도록 예시를 제공한다.

```text
예: 2026-05-어린이날
예: 2026-08-강릉여행
예: 2026-09-추석
```

### UX 요구사항

- 입력 필드는 크고 단순해야 한다.
- 최근에 만든 폴더명을 참고할 수 있으면 좋다.
- 폴더 생성 후 바로 사진 업로드로 이어질 수 있어야 한다.

예를 들어 다음 흐름을 지원한다.

```text
새 폴더 만들기
→ 2026-08-강릉여행 입력
→ 폴더 생성 완료
→ 이 폴더에 사진 올리기
```

## 🗺️ 지도 보기 요구사항

여행이나 외출 사진을 추억으로 보기 위해 지도 화면을 제공한다.  
KakaoMap API를 사용하여 가족이 다녀온 장소를 지도 위에 표시한다.

### 지도 기능의 목적

지도는 정확한 위치 관리 도구가 아니라, 가족이 다녀온 장소를 예쁘게 다시 보는 감상 기능이다.

### 지도에 표시할 정보

- 사진방 내 장소 폴더
- 여행 폴더
- 장소명이 포함된 폴더
- 사용자가 선택한 대표 장소

### 장소 정보 입력 방식

MVP에서는 사진 EXIF GPS 자동 분석을 필수로 하지 않는다.  
대신 폴더명 또는 사용자가 입력한 장소명을 기준으로 지도에 표시한다.

예시는 다음과 같다.

```text
2026-08-강릉여행 → 강릉
2026-10-부산가족모임 → 부산
2025-12-할머니집 → 사용자가 장소 등록 필요
```

### 지도 화면 구성

- 상단: 사진방 이름
- 중앙: KakaoMap 지도
- 지도 위: 장소 마커
- 하단: 장소별 앨범 카드

### 지도 카드 예시

```text
강릉
2026년 8월
사진 42장
```

카드를 누르면 해당 폴더 또는 앨범 화면으로 이동한다.

### 지도 UX 방향

- 관광 지도처럼 복잡하지 않아야 한다.
- 가족 여행 발자국을 보는 느낌이어야 한다.
- 마커는 단순 핀보다 감성적인 앨범 마커 형태가 좋다.
- 지도 하단에는 가로 스크롤 앨범 카드를 배치한다.

## 📱 모바일 UX 원칙

MVP는 모바일 사용성을 우선한다.

### 기본 원칙

- 주요 버튼은 화면 하단에 배치한다.
- 한 손 조작이 가능해야 한다.
- 업로드와 폴더 생성은 하단 시트 방식이 적합하다.
- 사진 선택은 모바일 기본 사진 선택기를 활용한다.
- 페이지 이동보다 부드러운 전환을 우선한다.
- 화면은 파일 관리 도구가 아니라 앨범 감상 도구처럼 보여야 한다.

### 하단 액션 영역

사진방 홈에는 다음 액션이 잘 보여야 한다.

```text
[사진 올리기] [폴더 만들기]
```

또는 플로팅 버튼을 누르면 다음 메뉴가 열린다.

```text
+ 사진 올리기
+ 새 폴더 만들기
+ 지도 보기
```

- 여행 기록집
- 따뜻한 갤러리

### 시각 요소

- 둥근 카드
- 넓은 여백
- 부드러운 그림자
- 사진 중심 레이아웃
- 짧은 감성 문구
- 과하지 않은 애니메이션
- 모바일에서 부드러운 스크롤

사이트 접속

### 가족이 사진을 올리는 흐름

```text
사이트 접속

→ 사진방 입장
→ 사진 올리기 선택
→ 폴더 선택
→ 휴대폰 사진 선택
→ 업로드 완료
→ 최근 사진에 반영
```

```text
사진방 입장

### 여행 사진을 지도에서 보는 흐름

```

## ✅ MVP 성공 기준

MVP는 다음 조건을 만족하면 성공으로 본다.

- 가족이 별도 설명 없이 6자리 번호로 입장할 수 있다.
- 모바일에서 사진 업로드가 어렵지 않다.
- 모바일에서 폴더 생성 후 바로 사진을 올릴 수 있다.
- 사진방 홈이 파일 목록처럼 보이지 않는다.
- 최근 사진을 봤을 때 가족 앨범 같은 느낌이 난다.
- 폴더 목차가 디자인된 앨범 목록처럼 보인다.
- 지도 화면에서 여행지와 사진 폴더를 자연스럽게 연결할 수 있다.

## 📌 최종 정의

가족사진관 MVP는 3개의 가족 사진방을 6자리 번호로 구분하고, 각 사진방에 연결된 Google Drive 폴더의 사진을 최근순으로 감성적으로 보여주는 모바일 중심 웹사이트이다.

사용자는 로그인 없이 사진방에 입장하여 사진을 보고, 폴더를 만들고, 휴대폰의 사진을 업로드할 수 있다. 사이트는 폴더 목록을 파일 탐색기처럼 보여주지 않고, 가족 앨범의 목차처럼 디자인하여 제공한다. 여행이나 장소 기반 추억은 KakaoMap을 활용해 지도 위에 표시한다.

> 핵심은 관리 기능이 아니라 공유와 감상이다.  
> 가족이 사진을 쉽게 올리고, 따뜻하게 다시 볼 수 있는 작은 온라인
